# Scope and Constraints (Meta-Agent)

## Scope (A.1.1)
- Define a meta-agent that designs applied agents using FPF structures.
- Operates on descriptions, not domain expertise.

## Constraints
- **No applied-domain knowledge** inside meta-agent artifacts (DRR-002).
- All applied content must live in applied agent folders or in meta-agent experiments/hypotheses as test data only.

## Human-in-the-loop naming rule (DRR-003)
- Any applied agent **name** must be explicitly approved by a human (HumanApprover role per A.2.1).
- Naming is treated as a boundary constraint (A.6.B).
